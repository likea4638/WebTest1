# 🚀 部署 Web 便当盒到 GitHub Pages

## 快速部署步骤

### 第一步：启用 GitHub Pages

1. 访问你的 GitHub 仓库：https://github.com/likea4638/WebTest1
2. 点击 **Settings**（设置）
3. 在左侧菜单找到 **Pages**
4. 在 **Source** 下拉菜单中选择 **GitHub Actions**
5. 保存设置

### 第二步：推送代码

**在哪里输入这些命令？**

**最简单的方法：**
1. 打开文件夹：`C:\Users\LiKe\Desktop\WebTest1`
2. 在文件夹地址栏（显示路径的地方）输入：`powershell` 然后按回车
3. 会打开一个命令行窗口，直接输入下面的命令即可

**或者：**
- 在文件夹中按住 `Shift` 键，右键点击空白处
- 选择 "在此处打开 PowerShell 窗口"

**然后执行以下命令：**

```bash
# 1. 切换到 main 分支（GitHub Actions 需要 main 分支）
git checkout main
# 如果提示分支不存在，使用：git checkout -b main

# 2. 添加所有文件
git add .

# 3. 提交更改
git commit -m "Add web box project with GitHub Pages deployment"

# 4. 推送到 GitHub
git push origin main
```

> 💡 **提示**：如果遇到问题，查看 `如何执行Git命令.md` 文件获取详细说明

### 第三步：等待自动部署

1. 在 GitHub 仓库页面，点击 **Actions** 标签
2. 你会看到 "Deploy to GitHub Pages" 工作流正在运行
3. 等待 2-3 分钟，部署完成后会显示绿色 ✓

### 第四步：访问网站

部署完成后，访问：
**https://likea4638.github.io/WebTest1/**

## 🔄 更新网站

每次修改代码后，只需：

```bash
git add .
git commit -m "Update web box"
git push origin main
```

GitHub Actions 会自动重新构建和部署！

## ❓ 常见问题

**Q: 部署后页面显示 404？**
A: 检查 GitHub Pages 设置中 Source 是否选择了 "GitHub Actions"

**Q: 样式或资源加载失败？**
A: 确保 `vite.config.js` 中的 `base` 路径设置为 `/WebTest1/`

**Q: 如何查看部署日志？**
A: 在 GitHub 仓库的 Actions 标签页查看工作流运行详情
